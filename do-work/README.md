# do-work

This folder is a file-based project management system used to plan, track, and execute features for this project. Each feature starts as a user request (UR), gets decomposed into discrete requirements (REQs), and is executed one at a time with a commit per task.

This project is built in public — these files are the decision log showing how each feature was planned and implemented.

## Structure

- `user-requests/` — Original feature briefs, recorded verbatim
- `working/` — The REQ currently being executed (max one at a time)
- `archive/` — Completed REQs with checked acceptance criteria
